# Implementation Plan: Agent Runner — Full System Specification

**Branch**: `001-full-system-spec` | **Date**: 2026-03-24 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-full-system-spec/spec.md`

## Summary

Consolidate the full agent-runner system (specs 001–006) into a single verified specification with comprehensive test coverage. The system is a Node.js server + Preact PWA that autonomously runs AI agent tasks, manages project lifecycles, streams real-time output, handles SSH agent bridging for git push via Yubikey, and provides an Android client. The codebase is already substantially built — this plan focuses on verification, gap-filling, and ensuring full test coverage per FR-082 through FR-096.

## Technical Context

**Language/Version**: TypeScript 5.9 on Node.js 22 (via Nix flake)
**Primary Dependencies**: `ws` (WebSocket), `web-push` (push notifications), `pino` (logging), `preact` (PWA client), `esbuild` (client bundler)
**Storage**: Filesystem — `~/.local/share/agent-runner/projects.json`, session metadata in `sessions/{id}/meta.json`, output in `sessions/{id}/output.jsonl`
**Testing**: Node.js native test runner (`tsx --test`), unit + integration + contract tests
**Target Platform**: Linux (NixOS) server, Android 8.0+ (API 26+) native client, any modern browser for PWA
**Project Type**: Web service + PWA + Android app
**Performance Goals**: Single-user, low-latency streaming (<100ms output delivery), sub-second API responses
**Constraints**: Single-user only, no auth, must run in Nix flake environment, `systemd-run --user` for sandboxing
**Scale/Scope**: <100 projects, <10 concurrent sessions, 7 PWA screens, 1 Android app

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

| Principle | Status | Notes |
|-----------|--------|-------|
| I. Sandbox-First Security | PASS | All agent processes sandboxed via `systemd-run --user`. Two-gate override for unsandboxed. SSH agent socket in allowed bind paths. |
| II. Markdown-as-Database | PASS | Task files are source of truth. Only `projects.json` as server state. Sessions are append-only artifacts. |
| III. Thin Client | PASS | PWA is read-only monitoring + input. All logic server-side. Reconnect replays from logs. |
| IV. NixOS-Native | PASS | All deps in `flake.nix`. Agents run via `nix develop`. No global installs. |
| V. Simplicity & YAGNI | PASS | No premature abstractions. Direct file I/O. Minimal dependencies. |
| VI. Process Isolation | PASS | Each session is independent with own log, metadata, lifecycle. Communication via filesystem only. |
| VII. Test-First | PASS | Test structure exists. This plan adds verification of completeness per FR-082–FR-096. |

**Post-design re-check**: All principles remain satisfied. No violations.
